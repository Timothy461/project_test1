# КОДИТОРГ — лендинг

Лендинг компании по сопровождению предпринимателей в госзакупках (44-ФЗ, 223-ФЗ). Next.js 14 + TypeScript + Tailwind CSS.

## Запуск локально

```bash
npm install
npm run dev
```

Откроется на `http://localhost:3000`.

## Структура

```
app/            — страница и layout (App Router)
components/     — LeadModal (форма заявки) и ErrorBoundary, подключены в app/page.tsx
lib/data.ts     — контент: услуги, статистика, отзывы, причины доверия
public/images/  — брендовые SVG-иллюстрации карточек услуг
```

## Особенности реализации

- Шрифт — Inter, подключён через `next/font/google` в `app/layout.tsx`.
- Цвета и поверхности заведены как токены Tailwind (`tailwind.config.ts`): `background`, `header`, `surface`, `panel`, `surface-alt`, `primary`, `accent`.
- Ширина контента унифицирована — все секции и шапка используют `max-w-7xl`.
- Мобильная навигация — выпадающее меню по кнопке-гамбургеру (пункты «Услуги», «Доверие», «Контакты» недостижимы на `<lg` без него).
- Кнопки, ссылки и поля формы — с видимым `focus-visible`-состоянием.
- Форма заявки (`components/lead-modal.tsx`) — с валидацией полей и `disabled`-состоянием кнопки во время отправки.
- Изображения карточек услуг — собственные SVG-иллюстрации в `public/images/` (в цветах `primary`/`accent`), без зависимости от внешних фотостоков. Фоновое фото hero-секции — `images.unsplash.com`.

Полный аудит текущей вёрстки (палитра, типографика, сетка, состояния, известные несоответствия) — в `design.md` в корне проекта. Более ранний черновой аудит статичного HTML-превью — в `docs/design.md`, само превью — в `preview/koditorg-landing-preview.html`.

## Что можно обсудить дальше

- Вынести повторяющиеся карточки (услуги/доверие/отзывы/контакты) в единый переиспользуемый компонент.
- Выровнять вертикальный ритм секции статистики (`py-12`) с остальными секциями (`py-20`/`py-32`), если это не осознанное решение.

## Деплой

Подойдёт любой хостинг, поддерживающий Next.js (Vercel, Netlify и т.п.).
