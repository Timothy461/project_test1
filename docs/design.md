# Design System — КОДИТОРГ

Лендинг для компании, сопровождающей предпринимателей по 44-ФЗ и 223-ФЗ. Документ описывает визуальную систему текущей реализации (`app/page.tsx`, `tailwind.config.ts`, `globals.css`) и фиксирует решения, принятые при сборке превью.

## Тон и характер

Тёмный, «официальный» интерфейс — холодные синие тона вместо нейтрального серого, чтобы отличаться от типичных юридических лендингов на белом фоне. Минимум декора, акцент на читаемость цифр и условий (цены, сроки, законы).

## Цветовая палитра

| Токен | HEX / значение | Где используется |
|---|---|---|
| `background` | `#0a1320` | Фон страницы |
| `card` | `#101b2b` / `#0f1a29` / `#0d1827` | Карточки услуг, модалка (разные секции — слегка разные оттенки) |
| `section-alt` | `#0c1625` | Фон секции «Контакты» |
| `foreground` | `#e5edf5` | Основной текст |
| `slate-400` | `#94a3b8` | Вторичный текст (описания) |
| `slate-500` | `#64748b` | Подписи, мелкий текст |
| `primary` | `#5aa0d6` | Кнопки, акцентные панели |
| `accent` | `#7dd3fc` | Иконки, hover-состояния, ссылки |
| `border` | `rgba(159,176,195,0.16)` | Тонкие разделители |
| `white/5`, `white/10` | `rgba(255,255,255,0.05 / 0.10)` | Подложки карточек, бордеры |

Фон страницы — не плоский цвет, а градиент: `radial-gradient(circle at top left, rgba(90,160,214,0.10), transparent 28%)` поверх `linear-gradient(180deg, #09111d → #0b1421)`.

## Типографика

- Шрифт: **Inter** (400–800), фолбэк Arial/sans-serif.
- H1: 30px на мобильном → 60px (md) → 72px (lg), вес 700, `line-height: 1.15`, `text-wrap: balance`.
- H2 секций: 30px → 48px.
- Текст-лид: 16px → 20px.
- Тело/описания: 14–16px.
- Лейблы/eyebrow (капс): минимум **12px** — 10px признан нечитаемым на мобильном и поднят при аудите.

## Сетка и отступы

- Контейнер: `max-width: 1280px`, паддинг 16px (мобильный) / 24px (md+).
- Брейкпоинты: 640 / 768 / 1024px (sm/md/lg, как в Tailwind по умолчанию).
- Секции: вертикальный паддинг 80px (мобильный) → 128px (md+).
- Карточки и блоки в сетке: gap 24–32px, на мобильном — всегда один столбец (многоколоночность только от md/lg), порядок в DOM уже mobile-first.
- Радиусы: крупные — 2rem (карточки, модалка, hero-панель), кнопки — `999px` (пилюля).

## Компоненты

- **Header** — фиксированный, высота 64px (мобильный) / 80px (md+); логотип + бейдж закона (скрыт на мобильном) + nav (только lg+) + CTA-кнопка.
- **Hero** — фоновое фото в grayscale 20% + градиент, заголовок/лид/CTA слева, панель «Кому помогаем» справа (на мобильном уходит вниз).
- **Stats** — 3 карточки, на мобильном в столбец.
- **Service cards** — фото 224px + заголовок + описание + цена + ссылка «Обсудить».
- **Contacts** — 2 плитки контактов + CTA-панель на фирменном цвете.
- **Lead modal** — оверлей с блюром, форма (имя + телефон/email) → состояние «заявка принята».
- **FAB** — плавающая кнопка Telegram, правый нижний угол.

## Минимальные требования для мобильной версии (375px)

Проверено реальным рендером в headless Chromium на 375×812:

| Элемент | Минимум | Факт после правок |
|---|---|---|
| CTA-кнопки (шапка, карточки, крестик модалки) | ≥44px по высоте | 44px |
| Базовый текст | без зума, ≥16px для тела | 16px |
| Лейблы/капс-текст | ≥12px | 12px (было 10px) |
| Карточки в сетке | без пересечений | подтверждено замером координат — зазор 32px |

## Известные проблемы в исходном коде проекта

Зафиксировано при сборке превью, не относится к самому дизайну, но важно при переносе в реальный Next.js-проект:

- `lib/data.ts` использовал `source.unsplash.com/random/...` — этот сервис Unsplash отключён с 2021 года. В превью заменено на прямые CDN-ссылки `images.unsplash.com/photo-...`.
- `trustPoints` и `testimonials` импортируются в `page.tsx`, но никогда не рендерятся — секция «Доверие» и отзывы фактически отсутствуют, хотя данные для них готовы.
- Пункт меню «Доверие» ссылается на `#trust` — такого id на странице нет, ссылка ведёт в никуда.
- `components/lead-modal.tsx` (с валидацией полей) и `components/error-boundary.tsx` (настоящий Error Boundary) написаны, но не используются — `page.tsx` определяет свои упрощённые версии локально.

## Открытые вопросы (не решены в превью)

- Нужна ли секция «Доверие» / отзывы — добавить рендер `trustPoints`/`testimonials` или удалить мёртвые данные и пункт меню.
- Какую версию модалки использовать в проде — простую (как в превью) или валидированную из `components/lead-modal.tsx`.
- Финальные изображения для карточек услуг — сейчас стоят тематически близкие, но не брендовые фото с Unsplash; для прода стоит заменить на реальные/заказные.
